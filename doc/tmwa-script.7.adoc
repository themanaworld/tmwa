== Name

tmwa-script - the TMWA server-side scripting language

== Synopsis

TMWA script source is plain text loaded by the map server (see
*tmwa-map*(1)).  Most scripts define NPCs, but the same language and the
same bytecode interpreter are used in several other contexts (see
*DESCRIPTION*).  A trivial NPC script looks like:

----
example.gat,30,40,0|script|Example NPC|123,{
    mes "Hello, world.";
    close;
}
----

== Description

TMWA has a single embedded scripting language.  Script source is compiled
to bytecode at server startup and the same interpreter
(`run_script` / `run_script_l`, see _src/map/script-call.cpp_) executes
that bytecode in every context below.  The language is therefore *not*
NPC-specific, even though scripted NPCs are by far the most common use.

A running script always has an *origin* (the NPC, item or other object the
bytecode belongs to) and may have an attached player, the *RID* (the
"running ID", the player a builtin like *mes* or *getitem* acts on).  Some
contexts attach a player automatically; others run with no player attached
and a builtin that needs one will fail unless the script first calls
*attachrid*.

=== Execution contexts

The interpreter is entered from the following places.  All of them run the
same language; the differences are only in what triggers the script and
whether a player is attached.

NPC dialogue and interaction::
A player clicking a scripted NPC runs that NPC's script body
(`npc_click`).  *next*, *menu*, *input* and similar builtins suspend the
script; the player's later replies resume it (`npc_scriptcont`).  The
clicking player is the RID.

NPC event labels::
Labels named `On<Something>` inside an NPC script are entry points
triggered by the server rather than by a click.  *donpcevent* and the
`callfunc`-style cross-NPC calls invoke them explicitly; the server
invokes them implicitly:
+
* `OnInit` runs for every NPC once, at server start.
* `OnClock<hhmm>`, `OnDay<...>`, `OnHour<...>`, `OnMinute<...>` and
  related labels run on a wall-clock schedule.
* `OnTimer<ms>` labels run on per-NPC timers
  (*initnpctimer*, *startnpctimer*).
* `OnTouch` runs when a player steps onto the NPC's touch area.
* `OnPCLoginEvent`, `OnPCLogoutEvent`, `OnPCDieEvent`,
  `OnPCKillEvent` and `OnMobKillEvent` run on the corresponding player
  or mob event; these are broadcast to every NPC defining the label.

Mob "death" / kill events::
A monster can be spawned with an associated NPC event (for example by
*monster* or *areamonster*).  When the monster dies the server runs that
event as an ordinary NPC event-label script, with the killing player as
RID.

Player-triggered timers::
*addtimer* schedules an NPC event to run for a specific player after a
delay; it executes as a normal event-label script when it fires.

Item scripts::
Items carry script fragments compiled by the same parser.  A *use*-type
item runs its `use_script` when consumed; equipment runs its
`equip_script` when equipped (and on recalculation of stats).  The
player using or wearing the item is the RID.  See _src/map/pc.cpp_.

The "magic" / spell system::
TMWA no longer has a separate spell language in the shipped server.  A
spell is just an NPC event registered with the *registercmd* builtin: a
chat message a player types is matched against the registered
invocations and, on a match, the associated NPC event runs as a normal
event-label script (`magic_message`, see _src/map/npc.cpp_).  The
historical standalone spell interpreter survives only under
_old-doc/old-spell-language_ and is not part of the running server.

User-defined functions and subroutines::
*callfunc* and *callsub* call, respectively, a named `function` script
object or a label in the current script.  These are not a separate
context: they push a return frame and keep running on the same
interpreter and the same RID.

NOTE: Atcommands (the `@`/`#` GM commands) are *not* a script context.
They are implemented in C++ in _src/map/atcommand.cpp_.  Scripts interact
with them only indirectly, for example through the *remotecmd* builtin.

=== Variable scopes

Variable names carry their scope in a sigil prefix; a trailing `$`
marks a string variable.

[cols="1,4"]
|===
| _name_ | NPC-scope variable, shared by the script's NPC
| `@`_name_ | temporary variable, attached to the RID for the current run
| `$`_name_ | global variable, shared server-wide and persisted
| `.`_name_ | function-scope (local) variable
| `.@`_name_ | function-scope temporary variable
|===

Names without a sigil that match a known player parameter (for example
`Hp`, `Zeny`, `BaseLevel`) read and write that parameter on the RID.

== Builtin functions

Each entry below is a single definition-list item: the term is the
builtin's signature, derived from the argument-signature string in the
`builtin_functions[]` table of _src/map/script-fun.cpp_, and the indented
text is its description.

In a signature the function name is shown in bold and each argument in
italics; an argument is named after the kind of value it expects.
Arguments inside `[ ]` are optional, and `...` marks a repeatable trailing
argument.  The argument-type vocabulary is:

[cols="1,4"]
|===
| _int_    | integer expression
| _str_    | string expression
| _expr_   | expression of any type
| _map_    | map name
| _x_, _y_ | map coordinates
| _label_  | a label in the current script
| _func_   | a function or label reference
| _var_    | the name of a variable or player parameter
| _item_   | an item (by numeric id or name)
| _player_ | a player (by name or account id)
| _event_  | an NPC event, written `NpcName::OnLabel`
| _timer_  | a timer interval, in milliseconds
| _mob_    | a monster species
| _status_ | a status-condition (status-change) id
|===

A `-> type` suffix shows the value the builtin pushes back as an
expression result; builtins with no suffix are statements.  These types
are intent only: the parser enforces argument *count* (and the optional /
variadic markers), not argument *type*.

//>>> GENERATED-BUILTIN-LIST
*mes*([_expr_])::
  Print a line of dialogue in the NPC message box of the attached player. With
  no argument an empty line is printed. The text stays on screen until *next*,
  *menu*, *close* or *clear* is reached.

*mesq*([_expr_])::
  Like *mes*, but the line is wrapped in double quotes before being sent. A
  convenience for quoting spoken text.

*mesn*([_expr_])::
  Like *mes*, but the line is wrapped in square brackets. With no argument the
  NPC's own visible name (the part before any `#`) is used, so `mesn;` prints
  a `[Name]` speaker header.

*clear*()::
  Clear the NPC dialogue window of the attached player without ending the
  script, so following *mes* lines start from a blank box.

*goto*(_label_)::
  Jump unconditionally to _label_, a label in the current script. Execution
  continues there.

*callfunc*(_func_)::
  Call the user-defined `function` object named by _func_ (a string). Control
  transfers to the function; *return* comes back here. This form passes no
  arguments; use *call* to pass arguments.

*call*(_func_[, _expr_...]) -> variant (may be absent)::
  Call a user-defined `function` (when _func_ is a string naming one) or a
  script position, passing the remaining arguments. The arguments are readable
  inside the callee with *getarg*. The value passed to *return* becomes this
  expression's result.

*callsub*(_label_)::
  Call a subroutine: transfer control to _label_ in the current script,
  pushing a return frame so a later *return* resumes after this call.

*getarg*(_int_[, _expr_]) -> variant (int or str)::
  Return argument number _int_ (0-based) passed to the current function or
  subroutine by *call* or *callsub*. If the index is out of range, the
  optional second argument is returned instead, or an ETX (0x03) marker string
  when no default is given.

*return*([_expr_])::
  Return from the current *callfunc*, *call* or *callsub*. If an argument is
  given it becomes the call's result value. Using *return* outside a call
  frame is deprecated and logs a warning.

*void*([_expr_...])::
  Evaluate and discard its arguments. Used to call an expression-valued
  builtin purely for its side effects.

*next*()::
  Suspend the script and show a "next" button in the player's dialogue window.
  The script resumes when the player clicks it.

*close*()::
  End the script and close the player's dialogue window. Using *close* inside
  a *callfunc*/*callsub* frame is deprecated.

*close2*()::
  Close the player's dialogue window but keep the script suspended rather than
  ending it, so execution can continue after the player dismisses the window.

*menu*(_str_, _label_...)::
  Show a selection menu to the player. Arguments come in pairs: a choice
  string and the _label_ to jump to if it is chosen. An empty choice string
  ends the displayed list. The script suspends; when the player picks an entry
  it jumps to the matching label, and the 1-based choice index is stored in
  `@menu`. Cancelling ends the script.

*rand*(_int_[, _expr_]) -> int::
  Return a random integer. With one argument, a value in the range 0..._int_-1
  (0 if _int_ is not positive). With two arguments, a value between the two
  bounds inclusive (the bounds are swapped if given out of order).

*isat*(_map_, _x_, _y_) -> int::
  Return 1 if the attached player is standing on map _map_ at exactly (_x_,
  _y_), 0 otherwise.

*warp*(_map_, _x_, _y_)::
  Teleport the attached player to map _map_ at (_x_, _y_). The warp is refused
  if the source map has the `nowarp` flag or the destination has `nowarpto`,
  unless the player is a GM of sufficient level.

*areawarp*(_map_, _x_, _y_, _x_, _y_, _map_, _x_, _y_)::
  Warp every player inside the rectangle (_x_,_y_) to (_x_,_y_) on the first
  map to the destination map at the final (_x_, _y_). The same
  `nowarp`/`nowarpto` GM-level restrictions as *warp* apply per player.

*heal*(_int_, _int_[, _expr_])::
  Restore _hp_ hit points and _sp_ spell points to the attached player
  (negative values subtract). A dead player healed for positive HP is
  resurrected. If the optional third argument is nonzero the healing is
  treated as item healing (subject to healing modifiers).

*injure*(_int_, _int_, _int_)::
  Deal _damage_ points of damage to the being whose id is the second argument,
  attributed to the being whose id is the first. The damage is shown to
  clients and may kill (and free) the target.

*input*(_var_)::
  Prompt the attached player for input and store the result in _var_. A string
  variable (name ending in `$`) opens a text prompt; a numeric variable opens
  a number prompt. The script suspends until the player answers. A negative
  numeric answer cancels the dialogue.

*requestitem*(_var_[, _expr_])::
  Prompt the attached player to choose items from their inventory and store
  the chosen item ids (or names, if _var_ is a string array) into the array
  _var_. The optional second argument is how many items to request (1 to 16,
  default 1). The script suspends until the player answers.

*requestlang*(_var_)::
  Prompt the attached player for their client language and store the answer in
  the string variable _var_. The script suspends until the client replies.

*if*(_int_, _func_...)::
  If _int_ is nonzero, call _func_ with the remaining arguments. Records the
  test result so a following *elif* or *else* can react to it.

*elif*(_int_, _func_...)::
  Continuation of an *if* chain: if no earlier branch was taken and _int_ is
  nonzero, call _func_ with the remaining arguments.

*else*(_func_...)::
  Continuation of an *if* chain: if no earlier *if*/*elif* branch was taken,
  call _func_ with the remaining arguments.

*set*(_var_, _expr_[, _expr_])::
  Assign _expr_ to the variable or player parameter _var_. The optional third
  argument selects whose copy of the variable to write: a character name or id
  for player-scope variables, an NPC name or id for NPC-scope (`.`) variables.

*get*(_var_, _expr_) -> variant (int or str)::
  Read the variable or player parameter _var_ belonging to the being named by
  _expr_ (a character name or id, or an NPC name or id for NPC-scope
  variables) and return its value. Returns 0 or an empty string when the
  target cannot be found.

*setarray*(_var_, _expr_...)::
  Fill the array _var_ starting at its given index with the listed values. For
  an NPC-scope (`.`) array the second argument is instead the NPC to target
  and the values follow.

*cleararray*(_var_, _expr_, _int_)::
  Set the first _int_ elements of array _var_ to the value _expr_.

*getarraysize*(_var_) -> int::
  Return the size of array _var_: one past the index of its last nonzero or
  non-empty element.

*getelementofarray*(_var_, _int_) -> variant (int or str)::
  Return element _int_ of the array _var_ as a variable reference, the runtime
  form of the `var[index]` subscript syntax. Indices outside 0...255 yield 0.

*array_search*(_expr_, _var_) -> int::
  Search array _var_ for the value _expr_ and return the index of the first
  match, or -1 if the value is not present.

*setlook*(_int_, _int_)::
  Change a cosmetic look value of the attached player. The first argument is a
  `LOOK` type (hair, weapon sprite, hair colour, and so on) and the second is
  the new value.

*countitem*(_item_) -> int::
  Return how many of item _item_ (by id or name) the attached player carries
  in their inventory.

*checkweight*(_item_, _int_) -> int::
  Return 1 if the attached player could carry _int_ more of item _item_
  without exceeding their weight limit, 0 otherwise (also 0 for an invalid
  item or non-positive amount).

*getitem*(_item_, _int_[, _expr_, _expr_])::
  Give _int_ units of item _item_ to a player. By default the recipient is the
  attached player; an optional fourth argument is a being id to give the item
  to instead (the third argument is unused). Items that do not fit in the
  inventory are dropped at the recipient's feet.

*makeitem*(_item_, _int_, _map_, _x_, _y_)::
  Create _int_ units of item _item_ on the ground of map _map_ at (_x_, _y_)
  as a floor item. The map name `this` places it on the attached player's
  current map.

*delitem*(_item_, _int_)::
  Remove _int_ units of item _item_ from the attached player's inventory.

*getcharid*(_int_[, _expr_]) -> int::
  Return an id of a character. The first argument selects which: 0 the
  character id, 1 the party id, 2 always 0 (guilds are unimplemented), 3 the
  account id. The optional second argument is a character name; without it the
  attached player is used. Returns -1 if no such player.

*getnpcid*([_expr_]) -> int::
  Return the block-list id of an NPC. With a name argument that NPC is looked
  up; otherwise the NPC running the script is used. Returns -1 if the NPC is
  not found.

*getversion*() -> int::
  Return the client-protocol version number reported by the attached player's
  client.

*strcharinfo*(_int_[, _expr_]) -> str::
  Return a string about a character: 0 the character name, 1 the party name, 2
  an empty string (guilds are unimplemented). The optional second argument is
  a being id; without it the attached player is used.

*getequipid*(_int_[, _expr_]) -> int::
  Return the item id worn in equipment slot _int_ (1 to 11, indexing the
  `equip_*` constants) of a player. The optional second argument is a
  character name; otherwise the attached player is used. Returns -1 if the
  slot is empty.

*bonus*(_int_, _int_)::
  Add an equipment-style stat bonus of value _int_ to the attached player. The
  first argument is an `SP` bonus type. Intended for use from item
  `equip_script` fragments; the effect lasts only until stats are
  recalculated.

*bonus2*(_int_, _int_, _int_)::
  Like *bonus*, but for bonus types that take an extra parameter (the second
  argument), with the value as the third argument.

*skill*(_int_, _int_[, _expr_])::
  Grant the skill whose id is the first argument at the level given by the
  second to the attached player. The optional third argument is a flag passed
  to the skill-granting code (default 1).

*setskill*(_int_, _int_)::
  Set the attached player's level in skill _int_ permanently to the second
  argument, clamped to 0...`MAX_SKILL_LEVEL`.

*getskilllv*(_int_) -> int::
  Return the attached player's current level in skill _int_ (0 if the player
  does not have it).

*overrideattack*([_expr_, _expr_, _expr_, _expr_, _expr_, _expr_])::
  Override the attached player's normal attack with a spell. The arguments are
  attack delay (ms), attack range, a status-change icon id, a weapon look item
  id, an NPC event to run on attack, and optionally a charge count (default
  1). Called with no arguments it removes the override and restores the normal
  attack.

*getgmlevel*() -> int::
  Return the GM privilege bitmask of the attached player (0 for an ordinary
  player).

*end*()::
  End the current script run. Using *end* inside a *callfunc*/*callsub* frame
  is deprecated.

*getopt2*() -> int::
  Return the attached player's `opt2` status-option bitmask.

*setopt2*(_int_)::
  Set the attached player's `opt2` status-option bitmask to _int_ and refresh
  the look and stats.

*savepoint*(_map_, _x_, _y_)::
  Set the attached player's respawn point to map _map_ at (_x_, _y_).

*gettimetick*(_int_) -> int::
  Return a time value. Type 0 is the server tick (an unsigned counter that
  wraps), type 1 is the number of seconds elapsed since midnight, type 2 is
  the Unix timestamp. Any other value behaves as type 0.

*gettime*(_int_) -> int::
  Return a component of the current local time. Type 1 second, 2 minute, 3
  hour, 4 weekday (0...6), 5 day of month, 6 month (1...12), 7 full year.
  Returns -1 for an unknown type.

*openstorage*()::
  Open the attached player's Kafra storage window and suspend the script.

*getexp*(_int_, _int_)::
  Grant _int_ base experience and _int_ job experience to the attached player.
  Negative amounts are ignored.

*mobinfo*(_int_, _int_) -> variant (int or str)::
  Return a field of the monster-database entry for species _int_. The second
  argument is a `MobInfo` selector (id, names, level, HP/SP, experience,
  attack and defence stats, attributes, element, mode, speeds, and drop
  id/name/percent for drop slots 0...9). Returns an int or a string depending
  on the field, or -1 for an unknown species or selector.

*mobinfo_droparrays*(_int_, _int_, _var_) -> int::
  Fill the array _var_ with the drops of monster species _int_. The second
  argument selects what to store: 0 item ids, 1 item names (string array), 2
  drop percents. Returns 0 on error, 1 if the monster has drops, 2 if it has
  none.

*getmobdrops*(_int_) -> int::
  Store the drops of monster species _int_ into the global arrays
  `$@MobDrop_item`, `$@MobDrop_name$` and `$@MobDrop_rate`, with the count in
  `$@MobDrop_count`. Returns 0 if the species is unknown, 1 if it has drops, 2
  if it has none.

*summon*(_map_, _x_, _y_, _str_, _str_, _mob_, _int_, _int_[, _expr_])::
  Spawn a single monster of species _mob_ owned by another being. The
  arguments are the map, x, y, the owner being id (passed as a string), a
  monster name, the species, an attitude code (servant, friendly, hostile,
  frozen), a lifespan in milliseconds, and optionally an NPC event to run on
  the monster's death.

*monster*(_map_, _x_, _y_, _str_, _mob_, _int_[, _expr_])::
  Spawn _int_ monsters of species _mob_ on map _map_ at (_x_, _y_) with the
  given monster name. The optional final argument is an NPC event run when
  each monster dies.

*areamonster*(_map_, _x_, _y_, _x_, _y_, _str_, _mob_, _int_[, _expr_])::
  Spawn _int_ monsters of species _mob_ scattered within the rectangle
  (_x_,_y_) to (_x_,_y_) on map _map_, with the given monster name. The
  optional final argument is an NPC event run when each monster dies.

*killmonster*(_map_, _event_)::
  Remove monsters on map _map_. With the special event string `All`, all
  non-permanently-spawned monsters are removed; otherwise only monsters whose
  death event matches _event_ are removed.

*donpcevent*(_event_)::
  Trigger the NPC _event_ (written `NpcName::OnLabel`) immediately, running it
  as an event-label script.

*addtimer*(_timer_, _event_[, _expr_])::
  Schedule the NPC _event_ to run for a player after _timer_ milliseconds. The
  optional third argument is a being id selecting the player; otherwise the
  attached player is used.

*addnpctimer*(_timer_, _event_)::
  Schedule the NPC _event_ to run after _timer_ milliseconds, attached to the
  NPC named in the event.

*initnpctimer*([_expr_])::
  Reset the NPC's timer to zero and start it running, which drives that NPC's
  `OnTimer<ms>` labels. The optional argument names another NPC to act on;
  otherwise the script's own NPC is used.

*startnpctimer*([_expr_])::
  Start (resume) the NPC timer without resetting it. The optional argument
  names another NPC to act on; otherwise the script's own NPC is used.

*stopnpctimer*([_expr_])::
  Stop the NPC timer. The optional argument names another NPC to act on;
  otherwise the script's own NPC is used.

*getnpctimer*(_int_[, _expr_]) -> int::
  Return information about an NPC timer: type 0 the current tick in
  milliseconds, 1 whether the timer is active, 2 the number of `OnTimer`
  events defined. The optional second argument names another NPC; otherwise
  the script's own NPC is used.

*setnpctimer*(_int_[, _expr_])::
  Set the NPC timer to _int_ milliseconds. The optional second argument names
  another NPC; otherwise the script's own NPC is used.

*setnpcdirection*(_int_, _int_, _int_[, _expr_])::
  Set the facing direction and sitting state shown for an NPC. The first
  argument is a direction, the second nonzero makes the NPC sit, and the third
  nonzero makes the change persistent rather than per-viewer. The optional
  fourth argument names another NPC; otherwise the script's own NPC is used.

*npcaction*(_int_[, _expr_, _expr_, _expr_])::
  Send a raw NPC client-action command to the attached player. The first
  argument is the command code; the remaining optional arguments are a target
  (a being id, or an NPC name when the command is 2) and x, y coordinates.

*camera*([_expr_, _expr_, _expr_])::
  Control the attached player's camera. With no arguments it returns the
  camera to the player. Two arguments move it to absolute coordinates (_x_,
  _y_). One or three arguments centre it on an actor (a being id, or the
  strings `rid`/`player`, `oid`/`npc`, `relative`, or an NPC name), optionally
  offset by (_x_, _y_).

*announce*(_str_, _int_)::
  Broadcast the message _str_. The flag _int_ controls scope and styling: with
  the low bits clear it is a server-wide announcement; with bit 0x08 set it is
  limited to the running NPC's surroundings, otherwise to the attached
  player's.

*mapannounce*(_map_, _str_, _int_)::
  Broadcast the message _str_ to every player on map _map_. The flag _int_
  controls styling.

*getusers*(_int_) -> int::
  Return a user count. Flag 1 returns the total number of players on the
  server; flag 0 is disabled (use *getmapusers*). Bit 0x08 selects the NPC
  rather than the player as the reference being.

*getmapusers*(_map_) -> int::
  Return the number of players currently on map _map_, or -1 if the map does
  not exist.

*getareausers*(_map_, _x_, _y_, _x_, _y_[, _expr_]) -> int::
  Return the number of players inside the rectangle (_x_,_y_) to (_x_,_y_) on
  map _map_. Hidden players are never counted; if the optional final argument
  is nonzero, dead players are also excluded. Returns -1 if the map does not
  exist.

*getareadropitem*(_map_, _x_, _y_, _x_, _y_, _int_[, _expr_]) -> int::
  Return the total quantity of floor items of type _item_ lying inside the
  rectangle (_x_,_y_) to (_x_,_y_) on map _map_. If the optional final
  argument is nonzero the matched floor items are also removed. Returns -1 if
  the map does not exist.

*enablenpc*(_str_)::
  Make the NPC named _str_ visible and interactive.

*disablenpc*(_str_)::
  Make the NPC named _str_ invisible and non-interactive.

*sc_start*(_int_, _status_, _int_[, _expr_])::
  Apply status condition _status_ to a being for _int_ milliseconds with value
  parameter _int_. By default the attached player is affected; the optional
  final argument is a being id to affect instead. For a few legacy
  potion-style conditions a duration under one second is reinterpreted as
  seconds.

*sc_end*(_int_[, _expr_])::
  Remove status condition _int_ from a being. By default the attached player
  is affected; the optional second argument is a being id instead.

*sc_check*(_int_[, _expr_]) -> int::
  Return nonzero if the status condition _int_ is currently active on a being.
  By default the attached player is checked; the optional second argument is a
  being id instead.

*debugmes*(_str_)::
  Print _str_, together with the current RID and OID, to the server log. For
  script debugging; nothing is shown to players.

*wgm*(_str_)::
  Send the message _str_, prefixed with `[GM]`, as a whisper to all online GMs
  of at least the configured hack-info GM level.

*gmlog*(_str_)::
  Write _str_, prefixed with `{SCRIPT}`, to the GM/atcommand audit log,
  attributed to the attached player.

*resetstatus*()::
  Reset the attached player's stat-point allocation, refunding spent points.

*attachrid*(_int_) -> int::
  Attach the player with being id _int_ to the script as the RID. Returns 1 if
  such a player exists (and is now attached), 0 otherwise.

*detachrid*()::
  Detach the current player (RID) from the script, so following
  player-specific builtins have no player to act on.

*isloggedin*(_int_) -> int::
  Return 1 if a player with being id _int_ is currently online, 0 otherwise.

*setmapflag*(_map_, _int_)::
  Set map flag number _int_ on map _map_.

*removemapflag*(_map_, _int_)::
  Clear map flag number _int_ on map _map_.

*getmapflag*(_map_, _int_) -> int::
  Return the state (0 or 1) of map flag number _int_ on map _map_, or -1 if
