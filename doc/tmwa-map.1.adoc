== Name

tmwa-map - The Mana World Athena map server

== Synopsis

*tmwa-map* [*--help*] [*--version*] [*--write-atcommand-config* _outfile_] [_config-file_...]

== Description

*tmwa-map* is the map server of TMWA (The Mana World Athena), an
eAthena-derived MMORPG server.  A complete TMWA installation runs three
cooperating daemons: the login server (*tmwa-login*(1)) authenticates
accounts, the char server (*tmwa-char*(1)) holds characters and routes
players, and the map server hosts the actual game world.  Several map
servers may connect to one char server.

The map server loads the world maps, spawns monsters, runs NPCs, and
hosts the scripting engine.  It compiles every script to bytecode at
startup and executes it with a single interpreter; see *tmwa-script*(7)
for the language and its builtin functions.

On startup the map server connects outward to the char server (whose
address and credentials come from the configuration) and listens for game
clients on its own port.  It keeps running until it receives *SIGTERM* or
*SIGINT*, or until a script calls the *mapexit* builtin.

The daemon refuses to run as root.

== Options

Options are processed left to right, interleaved with configuration file
arguments.

*--help*::
Print a usage summary and exit.

*--version*::
Print the server version and exit.

*--write-atcommand-config* _outfile_::
Write the default atcommand (GM command) configuration to _outfile_ and
exit without starting the server.  Use this to obtain a template you can
then edit.

_config-file_::
Path to a map-server configuration file (see *tmwa-map.conf*(5)).  Several
may be given and are loaded in order.  When no configuration file is
given, the server loads `conf/tmwa-map.conf` relative to the working
directory.

== Files

`conf/tmwa-map.conf`::
Default map-server configuration file, used when no _config-file_ argument
is given.  It selects the maps and NPC scripts to load and the link to the
char server.  See *tmwa-map.conf*(5).

The configuration also references the map data, the NPC script tree, and a
log file; their locations are set by configuration options rather than
fixed paths.

== Exit status

The map server exits with status 1 if a fatal error occurs during
startup (for example a configuration file that cannot be read or a bad
option), and 0 on a clean shutdown.
