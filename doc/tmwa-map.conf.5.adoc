== Name

tmwa-map.conf - configuration for the TMWA map server

== Description

This page documents the configuration of the TMWA map server,
*tmwa-map*(1).  For the line-oriented file syntax (comments,
_key_*:*_value_ lines, *import* and *version* directives) see
*tmwa-config*(5).

The file(s) named on the *tmwa-map* command line (or `conf/tmwa-map.conf`
by default) are *meta* configuration files.  They do not hold the map
options below directly; instead each key points at another file to load.
All of the keys below except *atcommand_conf* are required: there are no
built-in defaults, so omitting any of them aborts startup.  Use *import*
(see *tmwa-config*(5)) to compose the meta file from smaller pieces.

*map_conf*::
Path of the file holding the map-server options listed under
*MAP OPTIONS*.

*battle_conf*::
Path of the file holding the gameplay-tuning options; see
*tmwa-battle.conf*(5).

*atcommand_conf*::
Path of the atcommand (GM command) configuration file.  A template can be
produced with `tmwa-map --write-atcommand-config`.

*item_db*::
Path of the item database.

*mob_db*::
Path of the monster database.

*mob_skill_db*::
Path of the monster-skill database.

*skill_db*::
Path of the skill database.

*quest_db*::
Path of the quest database.

*const_db*::
Path of the script-constant database, mapping names to numbers for use in
scripts.

*resnametable*::
Path of the resource-name translation table.  Each line maps a logical
`.gat` map name (as named in *map:* directives) to the on-disk data
file (typically `.wlk`) that the map loader actually reads.

== Map options

These options appear in the file named by *map_conf*.  The default shown
is the value used when the option is omitted.

*map_ip* (default empty)::
Public address this map server advertises.

*map_port* (default `5121`)::
TCP port the map server listens on.  Must be at least 1024.

*char_ip* (default empty)::
Address of the char server to connect to.

*char_port* (default `6121`)::
Port of the char server.  Must be at least 1024.

*userid* (default empty)::
Account name this map server uses to authenticate to the char server.

*passwd* (default empty)::
Password matching *userid*.

*autosave_time* (default `1min`)::
Per-character save period: each online character is written to disk
once per interval, but the writes are spread out across the interval
(the autosave timer fires every `autosave_time / (online + 1)` and
flushes one character per tick), so save activity is effectively
continuous when many players are online.

*mapreg_txt* (default `save/mapreg.txt`)::
Path of the file storing persistent global script variables (the
`$`-prefixed registers).

*log_file* (default empty)::
Filename prefix of the map-server log file.  The actual file is
`<log_file>.<index>`; older files are rotated and gzipped over time.
Empty disables file logging.

*gm_log* (default empty)::
Filename prefix of the GM-action log file.  The actual file is
`<gm_log>.YYYY-MM`, rotated monthly.  Empty disables GM logging.

The following keys may appear *more than once*; each occurrence adds or
removes one entry, and they are processed in order:

*map*::
Add a `.gat` map to the set of maps the server loads.  The special
value `map: clear` removes every map currently in the set, which is
useful after an *import* to start from a clean slate.

*delmap*::
Remove a previously added map.  Useful after an *import* to drop a map
the imported file added.  The special value `delmap: all` clears the
entire map set in one step.

*npc*::
Add an NPC script file (or a script-tree root) to load.

*delnpc*::
Remove a previously added NPC script source.
